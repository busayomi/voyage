Route Listing and Descriptions

GET `/login`: Displays the login form for users.
POST `/login`: Handles user login. Validates credentials.
GET `/register`: Displays the registration form.
POST `/register`: Handles user registration, hashes password, saves user.
GET `/logout`: Logs out the user and ends their session.
GET `/profile`: Displays the userâ€™s profile and bookings (logged-in users only).

GET `/suites`: Displays the list of suites. Includes a search bar.
GET `/suites/:id`: Displays detailed info about a suite.

GET `/booking/:id`: Displays booking form for a specific suite. Prefills logged-in user's details.
POST `/submit-booking`: Submits booking info and saves it in the DB.

GET `/dining`: Displays menu items in categories like Appetizers, Main Courses, and Desserts.

GET `/events`: Lists upcoming and past events.
POST `/add-event`: Allows admin to add events with image upload.

GET `/bookings-and-requests`: Shows all bookings and contact requests (admin view).
GET `/view-bookings-requests`: Displays bookings and requests in a table format.
POST `/add-menu-item`: Admin can add a menu item with image.
POST `/add-suite`: Admin can add a new suite to the DB with images.
GET `/create-event`: Displays a form to create a new event.

Database Schema Overview

  • Users Table: Stores user data (ID, Name, Email, Password).
  • Suites Table: Stores suite details (Name, Description, Price, Images, etc.).
  • Bookings Table: Stores booking data (User info, Suite ID, Dates, Guests, etc.).
  • Menu Items Table: Stores menu item data (Name, Description, Price, Category, Image).
  • Events Table: Stores event data (Name, Date, Time, Description, Image).
  • Contact Requests Table: Stores contact form submissions.

Third-Party Packages

  • `express-session`: Manages user sessions.
  • `connect-flash`: Handles success and error flash messages.
  • `bcryptjs`: Encrypts passwords securely.
  • `multer`: Manages file uploads for menu items, suites, and events.
